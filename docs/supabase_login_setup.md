## Supabase 로그인 연동 사전 설정

현재 앱 구현은 `user_profile` 테이블에 `password_hash` 컬럼이 있다고 가정합니다.
비밀번호는 현재 `sha256:<digest>` 형식으로 저장됩니다.

### 1) 스키마 보완 SQL

```sql
-- 0. 먼저 id 컬럼 상태 확인
select column_name, data_type, is_nullable, column_default
from information_schema.columns
where table_schema = 'public'
	and table_name = 'user_profile'
	and column_name = 'id';

-- 1. 비밀번호 해시 컬럼 추가
alter table public.user_profile
add column if not exists password_hash text;

alter table public.user_profile
alter column password_hash set not null;

alter table public.user_profile
add constraint user_profile_nickname_unique unique (nickname);

-- 2-A. id가 integer/bigint 인 경우(둘 중 하나만 실행)
alter table public.user_profile
alter column id add generated by default as identity;

-- 2-B. id가 uuid 인 경우(둘 중 하나만 실행)
create extension if not exists pgcrypto;
alter table public.user_profile
alter column id set default gen_random_uuid();
```

> 기존 데이터가 있다면 `password_hash` 기본값 보정 후 `set not null`을 적용하세요.
> `2-A`와 `2-B`는 동시에 실행하지 말고, `id` 타입에 맞는 한 가지 방식만 적용하세요.

### 2) 최소 RLS 정책 예시 (MVP)

```sql
alter table public.user_profile enable row level security;

drop policy if exists "anon_can_lookup_nickname" on public.user_profile;
create policy "anon_can_lookup_nickname"
on public.user_profile
for select
to anon
using (true);

drop policy if exists "anon_can_insert_profile" on public.user_profile;
create policy "anon_can_insert_profile"
on public.user_profile
for insert
to anon
with check (true);
```

### 3) `invalid regular expression: invalid escape \\#` 오류 점검

아래 SQL로 `user_profile`의 제약식/정책을 먼저 확인하세요.

```sql
select conname, pg_get_constraintdef(c.oid) as definition
from pg_constraint c
join pg_class t on t.oid = c.conrelid
join pg_namespace n on n.oid = t.relnamespace
where n.nspname = 'public'
	and t.relname = 'user_profile';

select policyname, cmd, qual, with_check
from pg_policies
where schemaname = 'public'
	and tablename = 'user_profile';

-- 트리거 확인
select tgname, pg_get_triggerdef(t.oid) as trigger_def
from pg_trigger t
where t.tgrelid = 'public.user_profile'::regclass
	and not t.tgisinternal;

-- 함수 본문에서 '\\#' 검색 (aggregate/procedure 제외)
select n.nspname, p.proname
from pg_proc p
join pg_namespace n on n.oid = p.pronamespace
where p.prokind = 'f'
	and pg_get_functiondef(p.oid) like '%\\#%';
```

`\\#` 같은 escape가 들어간 정규식이 있으면 제거하거나 올바른 문자 클래스로 수정해야 합니다.

예시(정상):

```sql
-- 잘못된 예: nickname ~ '^[A-Za-z0-9\\#]+$'
-- 올바른 예: nickname ~ '^[A-Za-z0-9가-힣]+$'
```

### 4) `user_profile_nickname_format_chk` 제약 위반 해결

요구 규칙: `1~10자`, `한글/영문/숫자만 허용`, `특수기호 불가`.

```sql
alter table public.user_profile
drop constraint if exists user_profile_nickname_format_chk;

alter table public.user_profile
add constraint user_profile_nickname_format_chk
check (
	char_length(nickname) between 1 and 10
	and nickname ~ '^[A-Za-z0-9가-힣]+$'
);
```

### 5) `user_profile_id_fkey` 외래키 위반 해결

에러 예시:
`insert or update on table "user_profile" violates foreign key constraint "user_profile_id_fkey"`

현재 프로젝트는 Supabase Auth 회원 생성 없이 `user_profile`에 직접 가입하는 구조입니다.
이 경우 `user_profile.id -> users.id` 외래키가 있으면 신규 가입이 실패합니다.

```sql
-- 외래키 확인
select conname, pg_get_constraintdef(oid) as definition
from pg_constraint
where conrelid = 'public.user_profile'::regclass
	and contype = 'f';

-- 직접 가입 방식을 사용할 경우: FK 제거
alter table public.user_profile
drop constraint if exists user_profile_id_fkey;

-- id 자동 생성(uuid) 보장
create extension if not exists pgcrypto;
alter table public.user_profile
alter column id set default gen_random_uuid();
```

> 만약 Supabase Auth를 사용할 계획이라면 FK를 유지하고, 앱 로직을 `auth.signUp` 기반으로 바꿔야 합니다.
